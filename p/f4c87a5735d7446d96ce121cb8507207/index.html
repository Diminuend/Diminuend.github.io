打造坚不可摧的笔记后盾：Joplin WebDAV 加密笔记自动份至 GitHub 全指南

前言：为什么需要这样一套备份系统？

如果你在使用 Joplin 这款开源笔记软件，强烈建议开启 E2EE（端到端加密），并使用 WebDAV 进行多端同步。开启加密后，你的 WebDAV 服务器上存储的全是乱码命名的 .md 密文文件。

虽然这些密文无法直接阅读，但它们却是最完美的冷备份素材。只要你的主密码不泄露，即使把这些文件公之于众，也没有人能窥探你的隐私。

本文将教你如何利用 Docker + Rclone + Git，搭建一套轻量级的全自动流水线：
“从 WebDAV 拉取原始密文 -> 自动整理目录防止卡死 -> 增量推送到 GitHub 仓库”。

这套方案不依赖 Joplin 客户端，内存占用极低。即使哪天你的 WebDAV 服务商跑路了，你也能通过 GitHub 上的密文瞬间恢复所有笔记，实现数据的“狡兔三窟”。

第一步：准备 GitHub 仓库与访问令牌 (Token)

由于自动化脚本是在没有图形界面的服务器上运行的，我们需要一个专属的凭证来让脚本有权限向 GitHub 推送代码。

1. 创建私有仓库
登录 GitHub，新建一个仓库，命名为 joplin-cold-backup。建议设置为 Private (私有)，虽然存的是密文，但私有仓库总归更安心。不要勾选自动初始化 README 文件，保持仓库为空。

2. 获取 Personal Access Token (PAT)
用意：GitHub 现已废弃密码推代码的方式，必须使用 Token 才能在命令行完成身份验证。

  1. 点击 GitHub 右上角头像 -> Settings。
  2. 左侧栏拉到最底 -> Developer settings -> Personal access tokens -> Fine-grained tokens。
  3. 点击 Generate new token。
  4. 填写名称（如 Joplin-Backup），有效期建议选自定义（最长 1 年，到期需更换）。
  5. Repository access 选择 Only select repositories，并选中你刚才创建的 joplin-cold-backup。
  6. Permissions 中，找到 Contents，下拉选择 Read and write。
  7. 点击生成，立刻复制这串以 github_pat_ 开头的字符串并妥善保存。（刷新后将无法再次查看！）

第二步：配置 Rclone 连接 WebDAV

用意：Rclone 是云存储同步的“瑞士军刀”。它的多线程下载速度远超直接使用 Joplin 命令行，且支持增量同步，极大地节省了服务器带宽。

我们需要先生成一份 rclone.conf 配置文件。你可以在你的本地电脑或服务器上操作：

1. 生成配置
在终端输入 rclone config，按照以下交互步骤进行：

  1. 输入 n (新建远程连接)。
  2. name> 输入 my_webdav（名字要记住，后面脚本会用）。
  3. Storage> 寻找 webdav 并输入对应编号。
  4. url> 输入你的 WebDAV 地址（如坚果云的 https://dav.jianguoyun.com/dav/joplin）。
  5. vendor> 选择你的服务商，不知道就选 other。
  6. user> 输入 WebDAV 用户名。
  7. password> 输入 y，然后输入 WebDAV 密码。
  8. 一路回车保持默认，最后输入 q 退出。

2. 提取配置文件
输入 rclone config file，终端会显示配置文件的路径。打开该文件，将其中的内容复制出来，稍后我们要把它挂载到 Docker 里。

附录：常见错误与排坑指南 (Troubleshooting)

在实际搭建过程中，你可能会遇到以下典型报错。不用慌，对号入座即可解决：

🔴 错误 1：URL rejected: Port number was not a decimal number

现象：执行 git push 时报错，提示端口号不是十进制数字。
原因：你在配置 GH_REPO 变量时，把带有 @ 符号的邮箱（如 xxx@qq.com）当成了用户名。Git 会把第一个 @ 前面的内容当成账号，把 @ 后面的 qq.com: 误认为是服务器地址和端口号。
解决：在 URL 中不要使用邮箱。直接使用 Token 作为凭证：
https://github_pat_11A...@github.com/你的用户名/仓库名.git

🔴 错误 2：OpenSSL SSL_read: unexpected eof

现象：日志提示 Auto Backup 已提交，但在 Push 时报错意外结束 (EOF)。
原因：这是因为首次备份（或者你刚把几千个文件移动到子目录时），Git 需要推送的数据包巨大。受到国内连接 GitHub 网络的波动影响，数据传到一半超时断开了。
解决：

  1. 脚本中我已经加入了 git config --global http.postBuffer 524288000 来扩大缓冲区。
  2. 无需额外操作，多试几次即可。 因为 Git 支持断点续传和增量同步，只要最艰难的第一次（几千个文件）成功推上去，以后每天的日常备份只需推送变动的几个文件，网络压力极小，绝不会再报错。

🔴 错误 3：DNS: transient error (try again later)

现象：执行 docker compose up 时，卡在 apk add 步骤，提示 DNS 错误。
原因：宿主机的 Docker 网络无法解析 Alpine 镜像源的域名。

结语

至此，你的 Joplin 笔记已经拥有了最高等级的安全保障体系：

  • 隐私安全：E2EE 端到端加密，哪怕 GitHub 仓库设为公开，数据也不会泄露。
  • 容灾安全：NAS / WebDAV 存一份，本地存一份，GitHub 冷备份一份。
  • 零感体验：每天凌晨自动增量跑一次，不占资源，日志清晰。
